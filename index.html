Backend Platform Production Workflow

Full-Stack E-Commerce

A production-minded commerce system built with .NET, React, PostgreSQL, Redis, and Azure to show how I think about backend architecture, performance-sensitive reads, secure checkout, media workflows, and release quality in one coherent product.

  • Designed the system around a relational PostgreSQL core with Redis in front of high-frequency catalog reads so repeated product and listing requests stay fast instead of constantly hitting the database.
  • Implemented JWT authentication, Stripe checkout, and Cloudinary asset handling as separate concerns in the backend so identity, payments, and media each have a clean system boundary.
  • Used GitHub Actions to validate and deploy the application to Azure, which turns the project from a local demo into a more credible end-to-end engineering workflow.

Machine Learning Explainability

NFL QB Touchdown Predictor

A machine learning product that combines data preparation, XGBoost modeling, SHAP-based explainability, and a Streamlit delivery layer so the work is not just accurate, but also understandable and usable.

  • Trained the model on more than 10,000 historical plays, which gave the project enough scale to talk credibly about feature engineering, evaluation, and what the reported 88% accuracy actually represents.
  • Used SHAP explanations so predictions are tied back to the features influencing the result, which makes the model easier to defend in front of recruiters, teammates, or non-technical users.
  • Wrapped the pipeline in a Streamlit interface with a sub-two-second response target so the project feels like a lightweight product instead of a static notebook.
